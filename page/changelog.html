Список изменений YaMuTools

Текущая версия: 0.5.0

Обратная связь

Chimildic, 2021

Версия 0.7.5

  • Адаптация под Mozilla Firefox. Возможны баги, сообщите на форуме
  • Экспортер копирует в буфер обмена
  • Добавлена возможность удалить вкладку детям из навигации

Версия 0.7.3

  • К плееру радио добавлена кнопка повтора трека
  • Удалена гугл форма для обратной связи. Взамен форум GitHub.

Версия 0.7.2

  • В фильтре треков плейлиста появилась возможность удалить недавно игравшие треки (из истории прослушиваний).
  • К фильтру добавлена функция вычитания треков. Позволит удалить из текущего плейлиста то, что есть в других.

Версия 0.7.1

  • В меню плейлиста новая функция - фильтры треков. Позволит удалить из плейлиста треки: с пометкой "E", диз/лайки, кириллицу и ремиксы разной вариации, если указано в свойствах трека.
  • Исправлена ошибка при которой не скрывались элементы подкаста на главной странице. Яндекс сменил имена в разметке.
  • Исправлена ошибка при которой не работал фильтр ремиксов, если в плейлисте есть трек с отозванными правами.

Версия 0.6.16

  • Удалена настройка возврата вкладки Рекомендаций. Как и отмечалось ранее, она больше недоступна для перехода. Но пока доступны треки. Поэтому у Коллекционера пункт остался. Возможно скоро перестанет рабоать.
  • Добавлена сортировка треков плейлиста по алфавиту
  • Сокращен период появления напоминания оставить отзыв

Версия 0.6.14

  • В меню плейлиста новая функция - перемешать треки.
  • Исправлены незначительные ошибки
  • Яндекс.Деньги переведены на домен ЮMoney

Версия 0.6.8

  • Исправлена ошибка при которой плейлист "Топ прослушиваний" (last.fm) терял сортировку треков по количеству прослушиваний.

Версия 0.6.7

  • Новые плейлисты
    • Функция - Создать похожий плейлист. Доступно для любого плейлиста (находится в меню на странице плейлиста). Случайная подборка похожих треков, получаемых от Last.fm, проходящая фильтрацию порогом схожести и лимиту числа треков, очищается от дизлайков.
    • Забытое. В ленте рекомендий блок по 3 трека с описанием "Вы давно не слушали эти треки". Плейлист соберет все доступные блоки за выбранный период. Похоже на плейлист Тайник. Возможно ленту рекомендаций скоро уберут совсем. Уже сейчас наблюдается ее некорректная работа.
    • Из истории. Подборка из истории прослушиваний на Яндекс.Музыке. Отсекаются совсем недавние, явные дубликаты по id и дизлайки.
  • Теперь расширению не нужно разрешение на доступ к activeTab
  • Дополнены условия использования
  • Уведомление об обновлении расширения (включить в настройках).

Версия 0.6.6

  • Новый плейлист для блока Last.fm - Соседи. Подборка музыки, которую слушают пользователи Last.fm с близкими к вам вкусами.
  • Можно переназначить логин аккаунта Last.fm. По умолчанию он берется от привязки к Яндекс.Музыки. Но ручной ввод логина в настройках переопределит его. Появляется возможность получать плейлисты от любых пользователей Last.fm.
  • Расширение больше не требует прав на tabs.
  • Для плейлистов типа дискографии отключен механизм обновления/добавления треков. Каждый раз будет создаваться новый плейлист.
  • Инструмент подсчета статистики удален.
  • Изменена библиотека для всплывающих уведомлений.
  • Добавлены страницы: описания плейлистов, условия использования и обратной связи.

Версия 0.6.4

  • Доступен следующий выбор при создании плейлистов. Запоминается только последний созданный плейлист по его типу после активации настройки. При добавлении к существующему дубликаты не отслеживаются (их поиск запускается пользователем отдельно при необходимости).
    • Каждый раз новый плейлист
    • Обновить содержимое если существует
    • Добавить к уже существующему
    • Каждый раз спрашивать
  • Новая настройка - скрыть подкасты. Убирает ссылку на подкасты из шапки сайта, плейлист из персональной подборки, слайдер с подкастами. Блок "интересно сейчас" по-прежнему может показать подкаст. При отключении настройки требуется перезагрузка страницы, чтоб элементы вернулись.
  • В создаваемые плейлисты не добавляются треки с отметкой дизлайка. Исключение: плейлист "Мои дизлайки", плейлист с дискографией исполнителя.
  • Добавлена настройка выбора числа запросов к Last.fm. Влияет на количество треков в плейлистах: Рекомендовано, Микс, Скробблинг. Один запрос в среднем дает 25-30 треков. Last.fm может давать одинаковые треки в разных запросах, поэтому конечное общее число треков будет меньше.

Версия 0.5.6

  • Два новых плейлиста для блока Last.fm
    • Микс. Подборка музыки, которую скробблили ранее, а также новые рекомендации. Похоже на "Плейлист дня", только создается по требованию.
    • Скробблинг. Подборка только той музыки, которую скробблили ранее.

Версия 0.5.2

Новое

  • Эксперимент. Поиск дубликатов треков в плейлисте.
    • Создает копию плейлиста перед началом любой модификации.
    • Удаляет явные дубликаты (одинаковые id-треков).
    • Неявные дубли перемещает в начало списка для пользовательской модерации.
    • На время эксперимента функция не доступна в плейлисте "Мне нравится".
  • Случайная обложка при создании любого плейлиста. Можно отключить.
  • В меню загрузки обложки плейлиста добавлен пункт "Случайная обложка".

Правки

  • Исправлена ошибка, при которой расширение не работало в домене "ua".
  • Функции для плейлиста объединены в одно меню.
  • В настройках доступна кнопка для сброса по умолчанию.

Версия 0.5.0

Новое

  • Обладатели аккаунта Last.fm могут создавать плейлисты:
    • Топ прослушиваний. Чарт за выбранный период по числу прослушиваний.
    • Рекомендовано. Набор рекомендаций Last.fm. Каждая генерация - новые треки.
    • Любимые треки. Подборка всех треков с лайками, поставленных на Last.fm.
      Инструмент "Лайкер" поможет довести их до лайков Яндекс.Музыки.
  • Новые плейлисты у Коллекционера на базе ленты рекомендаций:
    • У поклонников. На базе блока "что в плейлистах у поклонников".
    • Редкие исполнители. Блок под названием "похоже, вам может понравиться".
    • Пропущено. Вы еще не слушали эти треки.
    • Недельный чарт. Собирает популярные треки недели за выбранный период.
  • Эксперимент: изменен механизм вставки элементов. Они появляются на странице быстрее. Получив сообщение об ошибке просьба сообщить по обратной связи.
  • Левый клик по иконке расширения открывает страницу настроек. Кроме того, переход к ней доступен в меню выбора пользователя (клик по иконке пользователя рядом с "Моя музыка").

Правки

  • Исправлена ошибка создания плейлиста "Мои дизлайки" с нулевым количеством треков.
  • Исправлена ошибка переадресации на отличный от авторизованного домена сайта после создания плейлиста.

Версия 0.4.0

Новое

  • Коллекционер научился собирать треки за определенный период. К примеру, дискографию исполнителя только за последние два года или новые релизы за 7 дней.
  • Настройка для Коллекционера. Убирать из подборки новых релизов неоригинальные версии треков. К примеру, mix или live. Можно отключить.
  • Настройка, дающая возможность убрать тизер плейлиста в правой верхней части сайта.
  • Новая верстка страниц расширения.

Правки

  • Крупный рефракторинг кода.
  • Устранена неисправность, по которой в набор новых релизов не попадали треки альбома при делении его на несколько дисков.
  • Исправлена ошибка перекрывания выпадающего меню нативными элементами сайта.
  • Пользовательская вкладка добавляется после вкладки "Радио", а не перед "Моя музыка".
  • Ссылки в инструкции "Экспортера" открываются в новой вкладке.
  • При первой установке автоматически открывается страница настроек.
  • Из описания плейлистов убрана строка про создание с помощью расширения.

Замечание

  • При обновлении с 0.3.2 до 0.4.0 настройки сбрасываются по умолчанию. В предыдущей версии не было механизма обработки обновления.

Версия 0.3.2

  • Новый инструмент "Экспортер". Простая реализация. Преобразует список треков в текст, который требуется передать сервису TuneMyMusic.
  • Инструмент "Коллекционер" научился собирать дискографию исполнителя.
  • Возможность открыть настройки расширения через меню выбора пользователя.
  • Подвал настроек получил ссылки на отзыв в Chrome Web Store и обратную связь.
  • Новая страница - список изменений.
